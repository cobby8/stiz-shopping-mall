## 2. 부족한 기능 (없는 것)

### 높은 우선순위 -- 운영에 직접 영향

| # | 기능 | 필요 이유 | 현재 상태 |
|---|------|----------|----------|
| H-1 | **고객 알림 (카카오/SMS)** | 상태 변경 시 고객에게 자동 알림이 안 감. 현재 카카오톡 수동 연락 중 | 알림 API 스텁만 존재 (POST /notify), 실제 발송 미구현 |
| H-2 | **결제 시스템 연동** | 고객이 온라인으로 결제할 수 없음. 현재 계좌이체 수동 확인 | PortOne 코드 껍데기만 있고 설정값 미입력 |
| H-3 | **주문 생성 UI (관리자)** | 관리자가 새 주문을 사이트에서 직접 생성하는 UI가 없음. API만 존재 | 복제/템플릿으로 우회 가능하나, 신규 고객 첫 주문 생성이 불편 |
| H-4 | **파일 첨부/관리** | 시안 파일, 디자인 확정본, 작업지시서를 시스템에서 직접 관리 불가 | designFileUrl/orderSheetUrl 필드는 있으나 업로드 기능 없음 (URL 수동 입력) |
| H-5 | **인쇄/견적서/작업지시서 출력** | 견적서, 작업지시서를 시스템에서 직접 인쇄할 수 없음 | 전혀 없음. 현재 별도 문서 작성 |

## 3. 보완할 기능 (있지만 개선 필요)

### 높은 우선순위

| # | 기능 | 현재 상태 | 개선 방향 |
|---|------|----------|----------|
| BH-1 | **마진 분석 신뢰도** | API 완성이나 원가 미입력 주문이 대부분이라 마진 데이터가 부정확 | 원가 입력 유도 (주문 상세에서 원가 입력 눈에 띄게 배치), 일괄 입력 기능 |
| BH-2 | **주문 상태 19단계 복잡도** | STATUS_FLOW가 19개이고, LEGACY_STATUS_MAP도 존재하여 관리가 복잡 | 실제 사용하는 상태만 정리하고, 레거시 매핑을 DB 마이그레이션으로 정리 |
| BH-3 | **미수금 관리** | 미수금 목록은 있으나, 입금 독촉 기능이나 미수금 기간별 분류가 없음 | 미수금 기간별 에이징(30일/60일/90일+) 추가, 독촉 알림 연계 |
| BH-4 | **파일 첨부 필드** | designFileUrl, orderSheetUrl, quoteUrl 필드가 있지만 URL 직접 입력만 가능 | 파일 업로드 UI + 서버 저장소 추가 (최소 로컬 파일 서빙) |
| BH-5 | **알림 발송 시스템** | POST /notify는 로그만 기록하는 스텁. "Phase 4에서 구현 예정"으로 보류 중 | 카카오 알림톡 API 연동 (비즈니스 채널 필요) 또는 최소 SMS |

### 중간 우선순위

| # | 기능 | 현재 상태 | 개선 방향 |
|---|------|----------|----------|
| BM-1 | **대시보드 홈 정보 밀도** | KPI 4개 + 파트별 대기량 + stale 주문만 표시 | 오늘 접수/납기 건수, 주간 추이, 최근 코멘트 등 추가 |
| BM-2 | **주문 상세 페이지 UX** | 모든 필드를 한 페이지에 표시하여 스크롤이 김 | 탭/아코디언으로 섹션 분리 (기본정보/디자인/결제/배송/이력) |

| BM-5 | **활동 로그 UI** | API는 있으나 프론트엔드에서 별도 조회 UI가 명확하지 않음 | 대시보드 홈 또는 별도 페이지에 활동 타임라인 위젯 |
| BM-6 | **주문 추적 페이지 접근성** | order-track.html은 주문번호를 알아야만 조회 가능 | 고객 로그인 후 본인 주문 자동 목록 표시 (myshop.html 연동) |
| BM-7 | **데이터 동기화 자동화** | sync-orders.js는 수동 실행 (CLI). 스프레드시트 변경 시 자동 반영 안 됨 | cron job 또는 Google Sheets API webhook 연동 |

## 4. 분석 관점별 평가

### 4-1. 운영 효율성 -- 평가: B+ (양호)

**잘 된 점:**
- 19단계 상태 관리 + 자동 이력 기록으로 주문 추적이 체계적
- 4파트 뷰로 팀별 업무 분리가 잘 되어 있음
- 주문 복제/템플릿으로 반복 작업 절감
- 일괄 상태 변경으로 대량 처리 가능
- 코멘트/태그로 팀 간 커뮤니케이션 보조

**부족한 점:**
- 고객 알림이 수동 (카톡 직접 연락)
- 파일 관리가 URL 복사 방식으로 비효율적
- 견적서/작업지시서 출력 기능 없음
- 주문 생성 UI가 없어 신규 주문 입력이 불편

### 4-2. 데이터 관리 -- 평가: A- (우수)

**잘 된 점:**
- SQLite로 전환 완료, 성능 개선
- 주문 상태 이력(order-history)이 자동 기록됨
- 활동 로그(activity-log)로 관리자 행동 추적
- 6시간 주기 자동 백업 + 수동 백업
- 고객 데이터가 별도 테이블로 분리됨

**부족한 점:**
- 원가 데이터 입력률이 낮아 마진 분석 불완전
- Excel/CSV 내보내기 없음
- 데이터 정합성 검증 도구 없음 (중복 주문번호 등)

### 4-3. 사용자 경험 (관리자) -- 평가: B (양호)

**잘 된 점:**
- 대시보드에 핵심 KPI 표시
- 필터/검색/정렬이 풍부
- 캘린더로 일정 시각화
- PC 최적화로 넓은 화면 활용

**부족한 점:**
- 모바일 사용 불가 (의도적 결정이나, 긴급 시 불편)
- 실시간 갱신 없음 (수동 새로고침)
- 주문 상세 페이지가 길어서 스크롤 필요
- 관리자 간 실시간 소통 기능 없음

### 4-4. 고객 경험 -- 평가: C (보통)

**잘 된 점:**
- 주문번호로 비로그인 추적 가능 (order-track.html)
- 4단계 프로그레스바로 직관적 상태 표시
- 상태 변경 타임라인 제공

**부족한 점:**
- 고객 알림이 전혀 자동화되지 않음
- 온라인 결제 불가
- 고객이 직접 커스텀 주문을 입력할 수 없음
- 마이페이지가 관리자 시스템과 완전 연동되지 않음
- 시안 확인/컨펌을 시스템에서 할 수 없음 (별도 채널 사용)

### 4-5. 비즈니스 인텔리전스 -- 평가: A- (우수)

**잘 된 점:**
- 연도별/월별/담당자별/종목별/고객별 매출 분석
- 매출 목표 설정 및 달성률 추적
- VIP/단골 자동 등급 분류
- 재주문 시기 도래 고객 자동 추출
- 미수금 고객별 집계

**부족한 점:**
- 마진 분석은 원가 입력 의존 (현재 입력률 낮음)
- 생산 리드타임 분석 부족 (평균 처리일은 있으나 병목 구간 분석 없음)
- 전년 대비 성장률 비교 차트 없음
- 예측 분석 (다음 달 예상 매출 등) 없음

### 4-6. 자동화/연동 -- 평가: D+ (미흡)

**잘 된 점:**
- 구글시트 -> DB 임포트/동기화 도구 구축 (수동 실행)
- 자동 백업 시스템
- 상태 변경 시 이력 자동 기록

**부족한 점:**
- 외부 시스템 연동 전무: 카페24, 결제사, 택배사, 카카오 알림톡 모두 미연동
- 시트 동기화가 CLI 수동 실행 (자동화 안 됨)
- 상태 변경 자동 알림 없음 (로그만 기록)
- 결제 확인 자동화 없음 (수동 입금 확인)

## 6. 핵심 인사이트

### 가장 잘 되어 있는 영역
- **주문 상태 관리**: 19단계 상태 + 4파트 뷰 + 이력 자동 기록. 구글 시트 수동 관리 대비 큰 진전.
- **매출 분석**: 7개 통계 API로 다각도 분석 가능. 대부분의 소규모 업체보다 충실.
- **고객 관리**: VIP 자동 분류, 재주문 후보, 고객 병합까지. CRM 수준.

### 가장 시급한 영역
- **고객 알림 자동화**: 현재 모든 고객 연락이 카카오톡 수동. 가장 큰 운영 병목.
- **파일 관리**: 시안, 작업지시서를 URL 복사로 관리하는 것은 비효율적. 파일 업로드가 시급.
- **주문 생성 UI**: API는 있는데 UI가 없어서, 새 주문을 넣으려면 기존 주문을 복제해야 함.

### 전체 완성도 평가
- 관리자 시스템 (백엔드): **80%** -- API/데이터 구조는 탄탄, 핵심 CRUD 완성
- 관리자 시스템 (프론트): **70%** -- 기능은 있으나 UX 세부 개선 필요
- 고객 대면 시스템: **30%** -- 주문 추적만 존재, 결제/알림/주문접수 미구현
- 외부 연동/자동화: **15%** -- 거의 모든 외부 연동이 미구현

> 요약하면, "관리자가 내부적으로 주문을 관리하는 시스템"으로서는 잘 갖춰져 있지만,
> "고객이 직접 사용하는 시스템"과 "외부 서비스 연동"은 아직 초기 단계이다.
> 단기적으로는 관리자 편의 기능(주문 생성 UI, 파일 업로드, Excel 내보내기)을 보강하고,
> 중기적으로 고객 알림(카카오)과 결제(PortOne) 연동에 집중하는 것이 효과적이다.
